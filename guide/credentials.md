# Credentials

Where every secret lives and how to rotate it. **No actual secret
values are in this file** — the source of truth for each is listed
in the "Lives in" column.

## Inventory

| Credential | Lives in | Rotate by |
| --- | --- | --- |
| **Server root password** | Contabo customer panel | Change in panel + run `passwd` over SSH; better, disable password auth entirely (see below) |
| **SSH private key** | Your laptop / password manager / [1Password SSH Agent](https://developer.1password.com/docs/ssh/) | Generate a new key, `ssh-copy-id` to the box, remove the old line from `/root/.ssh/authorized_keys` |
| **Cloudflare API token** | Cloudflare dashboard → My Profile → API Tokens | Revoke + create a new one with `Zone:DNS:Edit` on `shopifygmc.com` |
| **Postgres role passwords** | `/opt/gmcauditor/<env>/env/app.env` (`DATABASE_URL` line) | `sudo -u postgres psql -c "ALTER ROLE gmc_prod PASSWORD 'newpw'"`, then update env, restart |
| **`APP_SECRET`** | `/opt/gmcauditor/<env>/env/app.env` | `openssl rand -hex 32` → swap → restart. **Invalidates all CSRF tokens + signed unsubscribe links.** |
| **`SETTINGS_ENCRYPTION_KEY`** | `/opt/gmcauditor/<env>/env/app.env` | **Don't rotate without a re-encrypt step.** Encrypted refresh tokens in `store_gmc_connections` become unreadable; users have to re-consent to GMC. |
| **Platform admin password** | Postgres `users.password_hash` | <https://shopifygmc.com/account> → Change password (only if signed in via /login) — or via a `/admin` settings page (TBD) |
| **Mailbox passwords** | `/etc/dovecot/users` (ARGON2ID hashes) | Roundcube → Settings → Password (self-service), or `/admin/mail` Rotate button, or `mailbox passwd EMAIL` (CLI) |
| **DKIM private key** | `/etc/opendkim/keys/shopifygmc.com/mail.private` | `opendkim-genkey` with a new selector → publish new DKIM TXT in Cloudflare → swap selector in `/etc/opendkim.conf` → keep old TXT for a couple days then remove |
| **Gumroad webhook secret** | `/opt/gmcauditor/<env>/env/app.env` (`GUMROAD_WEBHOOK_SECRET`) | Rotate in Gumroad → update env → restart server |
| **Google OAuth client secret** | `/opt/gmcauditor/<env>/env/app.env` (`GOOGLE_OAUTH_CLIENT_SECRET`) | Rotate in Cloud Console → update env → restart server |
| **AI API key** | `platform_settings` table (encrypted with `SETTINGS_ENCRYPTION_KEY`) | <https://shopifygmc.com/admin/settings> |

## Server SSH

Right now there's a deploy SSH key in `/root/.ssh/authorized_keys` that
was generated by my Codespace and **disappears when that container is
gone**. You should:

1. Add your own key:
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/shopifygmc -C 'shopifygmc'   # if you don't have one
   ssh-copy-id -i ~/.ssh/shopifygmc.pub root@62.169.16.57
   ```

2. Verify your key works:
   ```bash
   ssh -i ~/.ssh/shopifygmc root@62.169.16.57 'whoami'
   ```

3. Then disable password auth:
   ```bash
   ssh -i ~/.ssh/shopifygmc root@62.169.16.57
   # On the box:
   sed -i 's/^#*PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
   sed -i 's/^#*PermitRootLogin.*/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
   systemctl restart ssh
   exit
   ```

4. (Optional) Remove the deploy key:
   ```bash
   ssh -i ~/.ssh/shopifygmc root@62.169.16.57 \
     'sed -i "/gmcauditor-deploy/d" /root/.ssh/authorized_keys'
   ```

## Contabo PTR record

This isn't really a credential, but it's the most-impactful unset
configuration item: outbound mail deliverability is meaningfully
worse until the reverse DNS for `62.169.16.57` is set to
`mail.shopifygmc.com`.

1. Log in to Contabo's customer panel.
2. Find your VPS in the list.
3. Look for "Reverse DNS" or "PTR Record" in the management menu.
4. Set it to `mail.shopifygmc.com`.
5. Wait ~10 minutes for global DNS to update.

Verify:
```bash
ssh root@62.169.16.57 'host 62.169.16.57'
# Should print: "57.16.169.62.in-addr.arpa domain name pointer mail.shopifygmc.com."
```

## What's NOT yet configured

These integrations are wired in code but unset in env, so the related
features will visibly say "not configured" until you fill them in:

- **Gumroad billing** — pricing page Upgrade buttons are visibly
  disabled until `GUMROAD_*` env vars + the 5 products in Gumroad's
  dashboard are set up. See README.md "Configuring Gumroad".

- **Google Merchant Center connect** — the Connect button shows an
  "OAuth not configured" message until `GOOGLE_OAUTH_CLIENT_ID`
  and `GOOGLE_OAUTH_CLIENT_SECRET` are set. See README.md
  "Connecting Google Merchant Center".

- **AI fix suggestions** — pipeline falls back to a deterministic
  mock until you set the AI API key from
  <https://shopifygmc.com/admin/settings>.
